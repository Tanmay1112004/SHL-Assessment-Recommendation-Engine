API Documentation

Integrate intelligent assessment matching into your existing systems

RESTful API Overview

Our API provides seamless integration capabilities for intelligent assessment recommendation. Built with modern REST principles, it offers fast, reliable access to our matching engine.

JSON-based request/response format
HTTPS-only for security
Rate limiting and authentication
Comprehensive error handling

Base URL

https://api.shl-assessment.com/v1

Authentication

Include your API key in the Authorization header:

Authorization: Bearer your-api-key

API Endpoints

Get Recommendations

POST

Request

POST /v1/recommend
Content-Type: application/json
Authorization: Bearer your-api-key

{
  "job_description": "string",
  "max_results": 10,
  "min_confidence": 0.3
}
Parameters
  • job_description (required): Full job description text
  • max_results (optional): Maximum number of recommendations (5-10)
  • min_confidence (optional): Minimum confidence threshold (0.0-1.0)

Response

{
  "success": true,
  "recommendations": [
    {
      "name": "Java Programming Test",
      "url": "https://www.shl.com/...",
      "confidence": 92.5,
      "type": "K",
      "duration": "60 minutes",
      "matched_skills": ["Java", "OOP"]
    }
  ],
  "processing_time": 1.23
}

List Assessments

GET

Request

GET /v1/assessments
Authorization: Bearer your-api-key

?type=K&limit=50&offset=0
Query Parameters
  • type (optional): Filter by test type (K or P)
  • limit (optional): Number of results per page (1-100)
  • offset (optional): Pagination offset

Response

{
  "success": true,
  "assessments": [
    {
      "name": "string",
      "description": "string",
      "test_type": "K|P",
      "skill_keywords": ["string"],
      "url": "string",
      "duration": "string",
      "difficulty": "string"
    }
  ],
  "total": 377,
  "page": 1,
  "per_page": 50
}

Batch Processing

POST

Request

POST /v1/batch
Content-Type: application/json
Authorization: Bearer your-api-key

{
  "job_descriptions": [
    "First job description...",
    "Second job description..."
  ],
  "max_results": 5
}

Process multiple job descriptions in a single request. Perfect for bulk operations.

Response

{
  "success": true,
  "results": [
    {
      "job_description": "string",
      "recommendations": [...]
    }
  ],
  "total_processed": 2
}

Error Handling

Error Response Format

{
  "success": false,
  "error": {
    "code": "INVALID_REQUEST",
    "message": "Job description is required",
    "details": {...}
  }
}

Common Error Codes

400 Invalid request parameters
401 Authentication required
403 Invalid API key
429 Rate limit exceeded
500 Internal server error

Integration Examples

JavaScript (Fetch)

const apiKey = 'your-api-key';
const jobDescription = `Your job description here...`;

fetch('https://api.shl-assessment.com/v1/recommend', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${apiKey}`
  },
  body: JSON.stringify({
    job_description: jobDescription,
    max_results: 10
  })
})
.then(response => response.json())
.then(data => console.log(data));

Python (Requests)

import requests

api_key = 'your-api-key'
job_description = '''Your job description here...'''

response = requests.post(
  'https://api.shl-assessment.com/v1/recommend',
  headers={
    'Content-Type': 'application/json',
    'Authorization': f'Bearer {api_key}'
  },
  json={
    'job_description': job_description,
    'max_results': 10
  }
)

results = response.json()
print(results)
